Bachelor's Final Project:
Mobile Application Development for Inovretail' SmartTrack

This project aims to develop a mobile application to be part of a Seeplus’s module, a product of Inovretail. The project grew out of a need to improve and facilitate this module’s installation and calibration process in the stores, the SmartTrack. The calibration process of this product is too laborious and time consuming so it was needed to make it more efficient.
Since the project is about developing a mobile application, there was a need to develop a series of previous work, called back end, which was the basis for the correct functioning of the application. This work involved the creation of various Web services (C#), analysis and configuration of some tables from a database and its connection procedures within the company server (SQL). Furthermore, it was necessary to make changes in the Raspberry Pi computers creating in them some Web services (Node.js) to make them possible to interact with the mobile application.
The mobile application was developed using the technology called NativeScript. It was chosen to be developed this way since when compiled at the end of its programming process, there is a possibility to generate executable files for multiple platforms such as Android and iOS, the operating systems that represent two of the major companies in the current smartphones market to Google and Apple, respectively.
Thus, this project doesn’t only involve a mobile application but also all the back end work that had to be developed for the successful operation of the mobile application.

  • For: Inovretail
  • Date: March 2015 - September 2015

Network Services Management of a Small Company

The main goal of this work was to install and configure a series of services in a network of a small business.

  • Linux gateway that isolates the company network from the "public" network
  • A Web server running on port 80 which should present a page identifying the company and accessible from the outside network from the gateway external address
  • Company's network separated in 2 VLANs: separation of wireless clients from ethernet clients
  • Wireless Clients use RADIUS to authenticate
  • Dynamic IPs assignment (DHCP)
  • DNS setup to assign names to each machine connected
  • VoIP service to make calls inside the company (Ekiga)
  • SNMP agent on gateway to monitor the traffic coming from the outside network

  • For: Instituto Superior de Engenharia do Porto
  • Date: September 2016 - January 2017
